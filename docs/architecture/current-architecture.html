Recruiting Pipeline — Proposed Local-First Architecture

Automation prepares, tracks, and proposes. The user owns data and approves every external action.

Truth and context

  • Obsidian stores editable career context.
  • Evidence ledger makes every résumé claim traceable.
  • Unknown metrics become questions, never inventions.

Safe automation

  • Read-only mail intake is narrow and auditable.
  • Structured local events can be reversible.
  • Fetched web text is evidence, not instructions.

Human authority

  • Review controls résumé changes and remote sync.
  • Job applications remain manually submitted.
  • All external side effects require approval.

Proposed architecture — local-first, credential-minimal, and open-source safe.
